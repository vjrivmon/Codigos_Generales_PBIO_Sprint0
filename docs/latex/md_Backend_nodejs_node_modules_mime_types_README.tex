The ultimate javascript content-\/type utility.

Similar to \href{https://www.npmjs.com/package/mime}{\texttt{ the {\ttfamily mime@1.\+x} module}}, except\+:


\begin{DoxyItemize}
\item {\bfseries{No fallbacks.}} Instead of naively returning the first available type, {\ttfamily mime-\/types} simply returns {\ttfamily false}, so do `var type = mime.\+lookup(\textquotesingle{}unrecognized') $\vert$$\vert$ \textquotesingle{}application/octet-\/stream\textquotesingle{}{\ttfamily .}
\item {\ttfamily No}new Mime(){\ttfamily business, so you could do}var lookup = require(\textquotesingle{}mime-\/types\textquotesingle{}).lookup{\ttfamily .}
\item {\ttfamily No}.define(){\ttfamily functionality}
\item {\ttfamily Bug fixes for}.lookup(path)\`{}
\end{DoxyItemize}

\hypertarget{md_Backend_nodejs_node_modules_mime_types_README_autotoc_md1976}{}\doxysubsection{mime.\+content\+Type(type)}\label{md_Backend_nodejs_node_modules_mime_types_README_autotoc_md1976}
Create a full content-\/type header given a content-\/type or extension. When given an extension, {\ttfamily mime.\+lookup} is used to get the matching content-\/type, otherwise the given content-\/type is used. Then if the content-\/type does not already have a {\ttfamily charset} parameter, {\ttfamily mime.\+charset} is used to get the default charset and add to the returned content-\/type.


\begin{DoxyCode}{0}
\DoxyCodeLine{mime.contentType('markdown') // 'text/x-\/markdown; charset=utf-\/8'}
\DoxyCodeLine{mime.contentType('file.json') // 'application/json; charset=utf-\/8'}
\DoxyCodeLine{mime.contentType('text/html') // 'text/html; charset=utf-\/8'}
\DoxyCodeLine{mime.contentType('text/html; charset=iso-\/8859-\/1') // 'text/html; charset=iso-\/8859-\/1'}
\DoxyCodeLine{}
\DoxyCodeLine{// from a full path}
\DoxyCodeLine{mime.contentType(path.extname('/path/to/file.json')) // 'application/json; charset=utf-\/8'}
