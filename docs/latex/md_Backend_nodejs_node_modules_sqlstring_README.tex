\hypertarget{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3048}{}\doxysubsection{Escaping query values}\label{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3048}
{\bfseries{Caution}} These methods of escaping values only works when the \href{https://dev.mysql.com/doc/refman/5.7/en/sql-mode.html\#sqlmode_no_backslash_escapes}{\texttt{ NO\+\_\+\+BACKSLASH\+\_\+\+ESCAPES}} SQL mode is disabled (which is the default state for My\+SQL servers).

In order to avoid SQL Injection attacks, you should always escape any user provided data before using it inside a SQL query. You can do so using the {\ttfamily Sql\+String.\+escape()} method\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId = 'some user provided value';}
\DoxyCodeLine{var sql    = 'SELECT * FROM users WHERE id = ' + SqlString.escape(userId);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM users WHERE id = 'some user provided value'}

\end{DoxyCode}


Alternatively, you can use {\ttfamily ?} characters as placeholders for values you would like to have escaped like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId = 1;}
\DoxyCodeLine{var sql    = SqlString.format('SELECT * FROM users WHERE id = ?', [userId]);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM users WHERE id = 1}

\end{DoxyCode}


Multiple placeholders are mapped to values in the same order as passed. For example, in the following query {\ttfamily foo} equals {\ttfamily a}, {\ttfamily bar} equals {\ttfamily b}, {\ttfamily baz} equals {\ttfamily c}, and {\ttfamily id} will be {\ttfamily user\+Id}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId = 1;}
\DoxyCodeLine{var sql    = SqlString.format('UPDATE users SET foo = ?, bar = ?, baz = ? WHERE id = ?',}
\DoxyCodeLine{  ['a', 'b', 'c', userId]);}
\DoxyCodeLine{console.log(sql); // UPDATE users SET foo = 'a', bar = 'b', baz = 'c' WHERE id = 1}

\end{DoxyCode}


This looks similar to prepared statements in My\+SQL, however it really just uses the same {\ttfamily Sql\+String.\+escape()} method internally.

{\bfseries{Caution}} This also differs from prepared statements in that all {\ttfamily ?} are replaced, even those contained in comments and strings.

Different value types are escaped differently, here is how\+:


\begin{DoxyItemize}
\item Numbers are left untouched
\item Booleans are converted to {\ttfamily true} / {\ttfamily false}
\item Date objects are converted to `\textquotesingle{}YYYY-\/mm-\/dd HH\+:ii\+:ss'{\ttfamily strings}
\item {\ttfamily Buffers are converted to hex strings, e.\+g.}X\textquotesingle{}0fa5\textquotesingle{}\`{}
\item Strings are safely escaped
\item Arrays are turned into list, e.\+g. `\mbox{[}\textquotesingle{}a', \textquotesingle{}b\textquotesingle{}\mbox{]}{\ttfamily turns into}\textquotesingle{}a\textquotesingle{}, \textquotesingle{}b\textquotesingle{}{\ttfamily }
\item {\ttfamily Nested arrays are turned into grouped lists (for bulk inserts), e.\+g.}\mbox{[}\mbox{[}\textquotesingle{}a\textquotesingle{}, \textquotesingle{}b\textquotesingle{}\mbox{]}, \mbox{[}\textquotesingle{}c\textquotesingle{}, \textquotesingle{}d\textquotesingle{}\mbox{]}\mbox{]}{\ttfamily turns into}(\textquotesingle{}a\textquotesingle{}, \textquotesingle{}b\textquotesingle{}), (\textquotesingle{}c\textquotesingle{}, \textquotesingle{}d\textquotesingle{}){\ttfamily }
\item {\ttfamily Objects that have a}to\+Sql\+String{\ttfamily method will have}.to\+Sql\+String()\`{} called and the returned value is used as the raw SQL.
\item Objects are turned into `key = \textquotesingle{}val'\`{} pairs for each enumerable property on the object. If the property\textquotesingle{}s value is a function, it is skipped; if the property\textquotesingle{}s value is an object, to\+String() is called on it and the returned value is used.
\item {\ttfamily undefined} / {\ttfamily null} are converted to {\ttfamily NULL}
\item {\ttfamily NaN} / {\ttfamily Infinity} are left as-\/is. My\+SQL does not support these, and trying to insert them as values will trigger My\+SQL errors until they implement support.
\end{DoxyItemize}

You may have noticed that this escaping allows you to do neat things like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var post  = \{id: 1, title: 'Hello MySQL'\};}
\DoxyCodeLine{var sql = SqlString.format('INSERT INTO posts SET ?', post);}
\DoxyCodeLine{console.log(sql); // INSERT INTO posts SET `id` = 1, `title` = 'Hello MySQL'}

\end{DoxyCode}


And the {\ttfamily to\+Sql\+String} method allows you to form complex queries with functions\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var CURRENT\_TIMESTAMP = \{ toSqlString: function() \{ return 'CURRENT\_TIMESTAMP()'; \} \};}
\DoxyCodeLine{var sql = SqlString.format('UPDATE posts SET modified = ? WHERE id = ?', [CURRENT\_TIMESTAMP, 42]);}
\DoxyCodeLine{console.log(sql); // UPDATE posts SET modified = CURRENT\_TIMESTAMP() WHERE id = 42}

\end{DoxyCode}


To generate objects with a {\ttfamily to\+Sql\+String} method, the {\ttfamily Sql\+String.\+raw()} method can be used. This creates an object that will be left un-\/touched when using in a {\ttfamily ?} placeholder, useful for using functions as dynamic values\+:

{\bfseries{Caution}} The string provided to {\ttfamily Sql\+String.\+raw()} will skip all escaping functions when used, so be careful when passing in unvalidated input.


\begin{DoxyCode}{0}
\DoxyCodeLine{var CURRENT\_TIMESTAMP = SqlString.raw('CURRENT\_TIMESTAMP()');}
\DoxyCodeLine{var sql = SqlString.format('UPDATE posts SET modified = ? WHERE id = ?', [CURRENT\_TIMESTAMP, 42]);}
\DoxyCodeLine{console.log(sql); // UPDATE posts SET modified = CURRENT\_TIMESTAMP() WHERE id = 42}

\end{DoxyCode}


If you feel the need to escape queries by yourself, you can also use the escaping function directly\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var sql = 'SELECT * FROM posts WHERE title=' + SqlString.escape('Hello MySQL');}
\DoxyCodeLine{console.log(sql); // SELECT * FROM posts WHERE title='Hello MySQL'}

\end{DoxyCode}
\hypertarget{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3049}{}\doxysubsection{Escaping query identifiers}\label{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3049}
If you can\textquotesingle{}t trust an SQL identifier (database / table / column name) because it is provided by a user, you should escape it with {\ttfamily Sql\+String.\+escape\+Id(identifier)} like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var sorter = 'date';}
\DoxyCodeLine{var sql    = 'SELECT * FROM posts ORDER BY ' + SqlString.escapeId(sorter);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM posts ORDER BY `date`}

\end{DoxyCode}


It also supports adding qualified identifiers. It will escape both parts.


\begin{DoxyCode}{0}
\DoxyCodeLine{var sorter = 'date';}
\DoxyCodeLine{var sql    = 'SELECT * FROM posts ORDER BY ' + SqlString.escapeId('posts.' + sorter);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM posts ORDER BY `posts`.`date`}

\end{DoxyCode}


If you do not want to treat {\ttfamily .} as qualified identifiers, you can set the second argument to {\ttfamily true} in order to keep the string as a literal identifier\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var sorter = 'date.2';}
\DoxyCodeLine{var sql    = 'SELECT * FROM posts ORDER BY ' + SqlString.escapeId(sorter, true);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM posts ORDER BY `date.2`}

\end{DoxyCode}


Alternatively, you can use {\ttfamily ??} characters as placeholders for identifiers you would like to have escaped like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId = 1;}
\DoxyCodeLine{var columns = ['username', 'email'];}
\DoxyCodeLine{var sql     = SqlString.format('SELECT ?? FROM ?? WHERE id = ?', [columns, 'users', userId]);}
\DoxyCodeLine{console.log(sql); // SELECT `username`, `email` FROM `users` WHERE id = 1}

\end{DoxyCode}
 {\bfseries{Please note that this last character sequence is experimental and syntax might change}}

When you pass an Object to {\ttfamily .escape()} or {\ttfamily .format()}, {\ttfamily .escape\+Id()} is used to avoid SQL injection in object keys.\hypertarget{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3050}{}\doxysubsection{Formatting queries}\label{md_Backend_nodejs_node_modules_sqlstring_README_autotoc_md3050}
You can use {\ttfamily Sql\+String.\+format} to prepare a query with multiple insertion points, utilizing the proper escaping for ids and values. A simple example of this follows\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId  = 1;}
\DoxyCodeLine{var inserts = ['users', 'id', userId];}
\DoxyCodeLine{var sql     = SqlString.format('SELECT * FROM ?? WHERE ?? = ?', inserts);}
\DoxyCodeLine{console.log(sql); // SELECT * FROM `users` WHERE `id` = 1}

\end{DoxyCode}


Following this you then have a valid, escaped query that you can then send to the database safely. This is useful if you are looking to prepare the query before actually sending it to the database. You also have the option (but are not required) to pass in {\ttfamily stringify\+Object} and {\ttfamily time\+Zone}, allowing you provide a custom means of turning objects into strings, as well as a location-\/specific/timezone-\/aware {\ttfamily Date}.

This can be further combined with the {\ttfamily Sql\+String.\+raw()} helper to generate SQL that includes My\+SQL functions as dynamic vales\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var userId = 1;}
\DoxyCodeLine{var data   = \{ email: 'foobar@example.com', modified: SqlString.raw('NOW()') \};}
\DoxyCodeLine{var sql    = SqlString.format('UPDATE ?? SET ? WHERE `id` = ?', ['users', data, userId]);}
\DoxyCodeLine{console.log(sql); // UPDATE `users` SET `email` = 'foobar@example.com', `modified` = NOW() WHERE `id` = 1}

\end{DoxyCode}
