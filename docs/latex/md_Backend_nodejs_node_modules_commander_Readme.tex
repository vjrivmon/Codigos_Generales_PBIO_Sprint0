\hypertarget{md_Backend_nodejs_node_modules_commander_Readme_autotoc_md499}{}\doxysection{Options}\label{md_Backend_nodejs_node_modules_commander_Readme_autotoc_md499}
Options are defined with the {\ttfamily .option()} method, also serving as documentation for the options. Each option can have a short flag (single character) and a long name, separated by a comma or space or vertical bar (\textquotesingle{}$\vert$\textquotesingle{}).

The options can be accessed as properties on the Command object. Multi-\/word options such as \char`\"{}-\/-\/template-\/engine\char`\"{} are camel-\/cased, becoming {\ttfamily program.\+template\+Engine} etc. See also optional new behaviour to avoid name clashes.

Multiple short flags may optionally be combined in a single argument following the dash\+: boolean flags, followed by a single option taking a value (possibly followed by the value). For example {\ttfamily -\/a -\/b -\/p 80} may be written as {\ttfamily -\/ab -\/p80} or even {\ttfamily -\/abp80}.

You can use {\ttfamily -\/-\/} to indicate the end of the options, and any remaining arguments will be used without being interpreted.

Options on the command line are not positional, and can be specified before or after other arguments.\hypertarget{md_Backend_nodejs_node_modules_commander_Readme_autotoc_md500}{}\doxysubsection{Common option types, boolean and value}\label{md_Backend_nodejs_node_modules_commander_Readme_autotoc_md500}
The two most used option types are a boolean option, and an option which takes its value from the following argument (declared with angle brackets like {\ttfamily -\/-\/expect \texorpdfstring{$<$}{<}value\texorpdfstring{$>$}{>}}). Both are {\ttfamily undefined} unless specified on command line. ~\newline
