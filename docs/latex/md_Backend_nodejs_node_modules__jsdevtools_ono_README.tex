The {\ttfamily format} option let you set a format function, which replaces placeholders in error messages with values.

When running in Node.\+js, Ono uses \href{https://nodejs.org/api/util.html\#util_util_format_format_args}{\texttt{ the {\ttfamily util.\+format()} function}} by default, which lets you use placeholders such as s, d, and j. You can provide the values for these placeholders when calling any Ono method\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{throw ono("{}\%s is invalid. Must be at least \%d characters."{}, username, minLength);}

\end{DoxyCode}


Of course, the above example could be accomplished using \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Template_literals}{\texttt{ ES6 template literals}} instead of format strings\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{throw ono(`\$\{username\} is invalid. Must be at least \$\{minLength\} characters.`);}

\end{DoxyCode}


Format strings are most useful when you don\textquotesingle{}t alrady know the values at the time that you\textquotesingle{}re writing the string. A common scenario is localization. Here\textquotesingle{}s a simplistic example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const errorMessages \{}

\DoxyCodeLine{    zh: "{}\%s 无效。 必须至少\%d个字符。"{},}
\DoxyCodeLine{  \}}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{let lang = getCurrentUsersLanguage();}
\DoxyCodeLine{}
\DoxyCodeLine{throw ono(errorMessages.invalidLength[lang], username, minLength);}

\end{DoxyCode}

There are two ways to use Ono with your own custom error classes. Ono has built-\/in support for all of \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Error\#Error_types}{\texttt{ the built-\/in Java\+Script Error types}}. All of these built-\/in Java\+Script Error types accept a single parameter\+: the error message string. If your own error classes also work this way, then you can create Ono methods for your custom error classes. Here\textquotesingle{}s an example\+:

\DoxyCodeLine{    this.id = ++counter;}
\DoxyCodeLine{    this.timestamp = new Date();}
\DoxyCodeLine{  \}}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{// Create a new Ono method for your custom Error class}
\DoxyCodeLine{ono.myError = new Ono(MyErrorClass);}
\DoxyCodeLine{}
\DoxyCodeLine{// You can use this method just like any other Ono method}
\DoxyCodeLine{throw ono.myError(\{ code: 404, status: "{}NOT\_FOUND"{} \}, `Resource not found: \$\{url\}`);}

If your custom error classes require more than just an error message string parameter, then you\textquotesingle{}ll need to use Ono differently. Rather than creating a custom Ono method and using {\ttfamily ono.\+my\+Error()} syntax, you\textquotesingle{}ll use Ono {\itshape inside} your error class\textquotesingle{}s constructor. This has a few benefits\+:


\begin{DoxyItemize}
\item Your error class can accept whatever parameters you want
\item Ono is encapsulated within your error class
\item You can use {\ttfamily new My\+Error()} syntax rather than {\ttfamily ono.\+my\+Error()} syntax
\end{DoxyItemize}


\begin{DoxyCode}{0}
