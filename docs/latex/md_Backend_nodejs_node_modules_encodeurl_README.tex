Encode a URL to a percent-\/encoded form, excluding already-\/encoded sequences.

This function accepts a URL and encodes all the non-\/\+URL code points (as UTF-\/8 byte sequences). It will not encode the \char`\"{}\%\char`\"{} character unless it is not part of a valid sequence ({\ttfamily \%20} will be left as-\/is, but {\ttfamily foo} will be encoded as {\ttfamily \%25foo}).

This encode is meant to be \char`\"{}safe\char`\"{} and does not throw errors. It will try as hard as it can to properly encode the given URL, including replacing any raw, unpaired surrogate pairs with the Unicode replacement character prior to encoding.\hypertarget{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md782}{}\doxysection{Examples}\label{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md782}

\hypertarget{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md785}{}\doxysection{Similarities}\label{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md785}
This function is {\itshape similar} to the intrinsic function {\ttfamily encode\+URI}. However, it will not encode\+:


\begin{DoxyItemize}
\item The {\ttfamily \textbackslash{}}, {\ttfamily $^\wedge$}, or {\ttfamily $\vert$} characters
\item The {\ttfamily \%} character when it\textquotesingle{}s part of a valid sequence
\item {\ttfamily \mbox{[}} and {\ttfamily \mbox{]}} (for IPv6 hostnames)
\item Replaces raw, unpaired surrogate pairs with the Unicode replacement character
\end{DoxyItemize}

As a result, the encoding aligns closely with the behavior in the \href{https://url.spec.whatwg.org/}{\texttt{ WHATWG URL specification}}. However, this package only encodes strings and does not do any URL parsing or formatting.

It is expected that any output from {\ttfamily new URL(url)} will not change when used with this package, as the output has already been encoded. Additionally, if we were to encode before {\ttfamily new URL(url)}, we do not expect the before and after encoded formats to be parsed any differently.\hypertarget{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md786}{}\doxysection{Testing}\label{md_Backend_nodejs_node_modules_encodeurl_README_autotoc_md786}
