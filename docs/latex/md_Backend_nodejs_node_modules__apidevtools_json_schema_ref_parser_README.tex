\hypertarget{md_Backend_nodejs_node_modules__apidevtools_json_schema_ref_parser_README_autotoc_md2}{}\doxysection{The Solution\+:}\label{md_Backend_nodejs_node_modules__apidevtools_json_schema_ref_parser_README_autotoc_md2}
JSON Schema \$\+Ref Parser is a full \href{https://tools.ietf.org/html/draft-pbryan-zyp-json-ref-03}{\texttt{ JSON Reference}} and \href{https://tools.ietf.org/html/rfc6901}{\texttt{ JSON Pointer}} implementation that crawls even the most complex \href{http://json-schema.org/latest/json-schema-core.html}{\texttt{ JSON Schemas}} and gives you simple, straightforward Java\+Script objects.


\begin{DoxyItemize}
\item Use {\bfseries{JSON}} or {\bfseries{YAML}} schemas --- or even a mix of both!
\item Supports {\ttfamily \$ref} pointers to external files and URLs, as well as \href{https://apitools.dev/json-schema-ref-parser/docs/plugins/resolvers.html}{\texttt{ custom sources}} such as databases
\item Can \href{https://apitools.dev/json-schema-ref-parser/docs/ref-parser.html\#bundlepath-options-callback}{\texttt{ bundle}} multiple files into a single schema that only has {\itshape internal} {\ttfamily \$ref} pointers
\item Can \href{https://apitools.dev/json-schema-ref-parser/docs/ref-parser.html\#dereferencepath-options-callback}{\texttt{ dereference}} your schema, producing a plain-\/old Java\+Script object that\textquotesingle{}s easy to work with
\item Supports \href{https://apitools.dev/json-schema-ref-parser/docs/\#circular-refs}{\texttt{ circular references}}, nested references, back-\/references, and cross-\/references between files
\item Maintains object reference equality --- {\ttfamily \$ref} pointers to the same value always resolve to the same object instance
\item Tested in Node v10, v12, \& v14, and all major web browsers on Windows, Mac, and Linux
\end{DoxyItemize}\hypertarget{md_Backend_nodejs_node_modules__apidevtools_json_schema_ref_parser_README_autotoc_md3}{}\doxysection{Example}\label{md_Backend_nodejs_node_modules__apidevtools_json_schema_ref_parser_README_autotoc_md3}
