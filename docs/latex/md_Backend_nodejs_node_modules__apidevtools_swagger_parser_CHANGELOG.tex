This is a {\bfseries{complete rewrite}} of Swagger Parser. Major changes include\+:

{\bfseries{Swagger 2.\+0 Compliant}}~\newline
 Previous versions of Swagger Parser were based on early drafts of Swagger 2.\+0, and were not compliant with \href{https://github.com/OAI/OpenAPI-Specification/blob/master/versions/2.0.md}{\texttt{ the final version of the spec}}. Swagger Parser v3.\+0 is now compliant with the final spec as well as related specs, such as \href{https://tools.ietf.org/html/draft-pbryan-zyp-json-ref-03}{\texttt{ JSON Reference}} and \href{https://tools.ietf.org/html/rfc6901}{\texttt{ JSON Pointer}}

{\bfseries{All-\/\+New API}}~\newline
 The old API only had a single method\+: {\ttfamily parse()}. But depending on which options you passed it, the method did {\itshape much} more than its name implied. The new API has separate methods to make things a bit more intuitive. The most commonly used will be \href{https://github.com/APIDevTools/swagger-parser/blob/master/docs/swagger-parser.md\#validateapi-options-callback}{\texttt{ {\ttfamily validate()}}}, \href{https://github.com/APIDevTools/swagger-parser/blob/master/docs/swagger-parser.md\#bundleapi-options-callback}{\texttt{ {\ttfamily bundle()}}}, and \href{https://github.com/APIDevTools/swagger-parser/blob/master/docs/swagger-parser.md\#dereferenceapi-options-callback}{\texttt{ {\ttfamily dereference()}}}. The \href{https://github.com/APIDevTools/swagger-parser/blob/master/docs/swagger-parser.md\#parseapi-options-callback}{\texttt{ {\ttfamily parse()}}} and \href{https://github.com/APIDevTools/swagger-parser/blob/master/docs/swagger-parser.md\#resolveapi-options-callback}{\texttt{ {\ttfamily resolve()}}} methods are also available, but these are mostly just for internal use by the other methods.

{\bfseries{Asynchronous API}}~\newline
 The old API was \char`\"{}asynchronous\char`\"{}, but it relied on global state, so it did not support multiple simultaneous operations. The new API is truly asynchronous and supports both \href{http://javascriptplayground.com/blog/2015/02/promises/}{\texttt{ ES6 Promises}} and Node-\/style callbacks.

{\bfseries{New JSON Schema Validator}}~\newline
 Swagger Parser has switched from \href{https://github.com/geraintluff/tv4}{\texttt{ tv4}} to \href{https://github.com/zaggino/z-schema}{\texttt{ Z-\/\+Schema}}, which is \href{https://rawgit.com/zaggino/z-schema/master/benchmark/results.html}{\texttt{ faster}} and performs \href{https://github.com/ebdrup/json-schema-benchmark\#test-failure-summary}{\texttt{ more accurate validation}}. This means that some APIs that previously passed validation will now fail. But that\textquotesingle{}s a {\itshape good} thing!
