{\ttfamily yaml} is a Java\+Script parser and stringifier for \href{http://yaml.org/}{\texttt{ YAML}}, a human friendly data serialization standard. It supports both parsing and stringifying data using all versions of YAML, along with all common data schemas. As a particularly distinguishing feature, {\ttfamily yaml} fully supports reading and writing comments and blank lines in YAML documents.

The library is released under the ISC open source license, and the code is \href{https://github.com/eemeli/yaml/}{\texttt{ available on Git\+Hub}}. It has no external dependencies and runs on Node.\+js 6 and later, and in browsers from IE 11 upwards.

For the purposes of versioning, any changes that break any of the endpoints or APIs documented here will be considered semver-\/major breaking changes. Undocumented library internals may change between minor versions, and previous APIs may be deprecated (but not removed).

For more information, see the project\textquotesingle{}s documentation site\+: \href{https://eemeli.org/yaml/}{\texttt{ {\bfseries{eemeli.\+org/yaml}}}}

The API provided by {\ttfamily yaml} has three layers, depending on how deep you need to go\+: \href{https://eemeli.org/yaml/\#parse-amp-stringify}{\texttt{ Parse \& Stringify}}, \href{https://eemeli.org/yaml/\#documents}{\texttt{ Documents}}, and the \href{https://eemeli.org/yaml/\#cst-parser}{\texttt{ CST Parser}}. The first has the simplest API and \char`\"{}just works\char`\"{}, the second gets you all the bells and whistles supported by the library along with a decent \href{https://eemeli.org/yaml/\#content-nodes}{\texttt{ AST}}, and the third is the closest to YAML source, making it fast, raw, and crude.
